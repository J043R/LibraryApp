# Онлайн-библиотека

Клиент-серверное приложение на NestJS, SQLite и обычном
HTML/CSS/JavaScript. Пользователи, книги, выдачи и журнал сохраняются
в файловой базе данных между перезапусками сервера.

## Запуск

```powershell
cd .\backend
npm install
npm run build
npm start
```

После запуска приложение доступно по адресу `http://localhost:3000`.
Если порт занят, можно выбрать другой:

```powershell
$env:PORT=3001
npm start
```

Для проекта требуется Node.js 22.5 или новее. Дополнительно устанавливать
SQLite не нужно: используется встроенный модуль `node:sqlite`.

## База данных

Файл создается автоматически при первом запуске:

```text
backend/data/library.sqlite
```

В базе четыре таблицы:

| Таблица | Назначение |
|---|---|
| `users` | учетные записи, пароль и роль |
| `books` | книги библиотечного фонда |
| `borrowed_books` | выдачи, читатели и даты возврата |
| `logs` | журнал действий |

Роль хранится в поле `users.role`. Допустимые значения ограничены на уровне
SQLite: `admin`, `librarian`, `reader`, `guest`.

## Тестовые учетные записи

| Категория | Email | Пароль |
|---|---|---|
| Администратор | `bodrova@example.com` | `Admin123` |
| Библиотекарь | `ladygin@example.com` | `Librarian123` |
| Читатель | `ivan@example.com` | `Reader123` |

Администратор управляет пользователями и видит журнал. Библиотекарь видит
пользователей и управляет выдачей книг. Читатель видит свой профиль и может
брать и возвращать книги.

## Обязательные API

Все маршруты имеют префикс `/api`.

```text
GET  /api/users
GET  /api/users/:id
POST /api/users
```

Для защищенных запросов сначала выполняется `POST /api/auth/login`, затем
токен передается в заголовке `Authorization: Bearer <token>`.

Пример создания пользователя:

```json
{
  "name": "Анна Петрова",
  "email": "anna@example.com",
  "age": 20
}
```

`name` и `email` обязательны, `id` формируется сервером, `age` можно не
передавать. Email проверяется библиотекой `class-validator`.

## Структура

```text
backend/src/
  auth/       авторизация
  users/      обязательное API пользователей
  books/      каталог, выдача и возврат
  logs/       журнал действий
  security/   токены, роли, guard и декоратор @Roles
  database/   SQLite provider, схема и начальные данные
frontend/
  *.html      страницы приложения
  css/        оформление
  js/         Fetch API, роли и LocalStorage
```

Подробная привязка реализации к критериям находится в `CRITERIA.md`.
